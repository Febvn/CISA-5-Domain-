::D1_Q01:: Seorang auditor IS sedang merencanakan audit sistem informasi. Manakah dari berikut ini yang merupakan langkah pertama dan paling penting dalam mengembangkan strategi audit? {
    =Memahami tujuan bisnis dan risiko TI organisasi. #Benar. Strategi audit harus selaras dengan tujuan bisnis untuk memberikan nilai tambah.
    ~Menentukan sampel yang akan diuji. #Salah. Penentuan sampel adalah bagian dari pelaksanaan, bukan langkah strategis awal.
    ~Meminta akses ke semua sistem operasional. #Salah. Akses diberikan setelah ruang lingkup dan perencanaan final disepakati.
    ~Mengundang manajemen untuk pertemuan pembukaan. #Salah. Pertemuan pembukaan (entry meeting) dilakukan setelah rencana audit selesai disusun.
}

::D1_Q02:: Apa tujuan utama dari 'Audit Charter' (Piagam Audit)? {
    =Memberikan wewenang, tanggung jawab, dan akuntabilitas fungsi audit TI. #Benar. Audit Charter mendefinisikan peran dan legitimasi auditor di dalam organisasi.
    ~Menyusun jadwal audit tahunan yang detail. #Salah. Jadwal audit adalah bagian dari perencanaan operasional, bukan piagam.
    ~Menentukan standar audit yang akan digunakan. #Salah. Standar biasanya mengacu pada kerangka kerja eksternal (seperti ISACA), Piagam fokus pada wewenang internal.
    ~Mendokumentasikan hasil temuan audit sebelumnya. #Salah. Ini adalah laporan audit, bukan piagam.
}

::D1_Q03:: Dalam konteks audit TI, apa perbedaan utama antara 'Compliance Testing' dan 'Substantive Testing'? {
    =Compliance testing memeriksa kepatuhan pada prosedur kontrol, sedangkan Substantive testing memeriksa integritas data transaksi. #Benar. Compliance fokus pada "apakah aturan diikuti?", Substantive fokus pada "apakah datanya benar?".
    ~Compliance testing untuk data keuangan, Substantive testing untuk data operasional. #Salah. Keduanya bisa diterapkan di berbagai area.
    ~Compliance testing menggunakan CAATs, Substantive testing menggunakan wawancara. #Salah. Keduanya bisa menggunakan berbagai teknik termasuk CAATs.
    ~Compliance testing dilakukan oleh audit internal, Substantive testing oleh audit eksternal. #Salah. Kedua jenis audit dapat melakukan kedua pengujian tersebut.
}

::D1_Q04:: Manakah dari berikut ini yang BUKAN merupakan komponen dari 'Audit Risk'? {
    =Control Risk #Salah. Control Risk adalah risiko bahwa kontrol internal tidak mencegah atau mendeteksi kesalahan material. Ini BAGIAN dari Audit Risk.
    =Inherent Risk #Salah. Inherent Risk adalah kerentanan suatu aset tanpa memperhitungkan kontrol. Ini BAGIAN dari Audit Risk.
    =Detection Risk #Salah. Detection Risk adalah risiko auditor gagal mendeteksi kesalahan. Ini BAGIAN dari Audit Risk.
    ~Business Risk #Benar. Business Risk adalah risiko yang dihadapi organisasi (seperti kompetisi atau pasar), bukan risiko yang melekat pada proses audit itu sendiri.
}

::D1_Q05:: Seorang auditor menemukan bahwa staf TI tidak memisahkan tugas (Segregation of Duties/SoD) antara programmer dan operator komputer. Apa risiko utama kondisi ini? {
    =Programmer dapat memodifikasi kode program dan menjalankannya sendiri untuk melakukan kecurangan. #Benar. Kombinasi pemrograman dan operasi menciptakan peluang fraud yang tinggi karena tidak ada pengawasan.
    ~Sistem akan berjalan lebih lambat karena beban kerja programmer bertambah. #Salah. Ini adalah isu performa, bukan risiko kontrol utama dari SoD.
    ~Operator komputer tidak bisa memperbaiki bug dengan cepat. #Salah. Justru sebaliknya, operator tidak boleh sembarangan memodifikasi program.
    ~Backup data tidak akan dapat dilakukan secara otomatis. #Salah. SoD tidak berhubungan langsung dengan otomatisasi backup.
}

::D1_Q06:: Teknik sampling mana yang paling tepat digunakan ketika auditor yakin bahwa populasi data tidak mengandung kesalahan atau auditor ingin menguji transaksi spesifik bernilai tinggi? {
    =Judgmental Sampling (Non-statistical). #Benar. Auditor menggunakan penilaian profesional untuk memilih item berdasarkan risiko atau nilai material, bukan rumus statistik.
    ~Statistical Sampling. #Salah. Statistical sampling digunakan ketika auditor ingin memproyeksikan hasil sampel ke seluruh populasi dengan tingkat kepercayaan tertentu.
    ~Discovery Sampling. #Salah. Discovery sampling khusus digunakan untuk menemukan fraud atau kesalahan langka yang tidak boleh terjadi.
    ~Random Sampling. #Salah. Ini adalah metode pemilihan, bukan pendekatan sampling itu sendiri yang menentukan logika pengujian.
}

::D1_Q07:: Apa fungsi utama dari 'Computer Assisted Audit Techniques' (CAATs) seperti ACL atau IDEA? {
    =Menganalisis seluruh populasi data secara cepat dan mengidentifikasi anomali. #Benar. CAATs memungkinkan auditor memproses data besar (Big Data) yang tidak mungkin dilakukan manual.
    ~Menggantikan peran auditor dalam membuat keputusan akhir. #Salah. CAATs adalah alat bantu, auditor tetap yang menganalisis hasil dan mengambil keputusan.
    ~Membuat program perbaikan untuk bug sistem. #Salah. Ini adalah tugas developer, bukan fungsi CAATs.
    ~Mengotomatisasi pengiriman laporan audit ke manajemen. #Salah. Itu adalah fungsi sistem pelaporan atau email, bukan fitur utama CAATs untuk analisis data.
}

::D1_Q08:: Dalam kerangka kerja ISACA, mana yang merupakan siklus hidup laporan audit yang benar? {
    =Perencanaan -> Pelaksanaan -> Pelaporan -> Tindak Lanjut. #Benar. Ini adalah urutan standar proses audit.
    ~Pelaporan -> Perencanaan -> Pelaksanaan -> Tindak Lanjut. #Salah. Perencanaan harus duluan sebelum pelaporan.
    ~Pelaksanaan -> Perencanaan -> Tindak Lanjut -> Pelaporan. #Salah. Urutannya kacau.
    ~Tindak Lanjut -> Perencanaan -> Pelaporan -> Pelaksanaan. #Salah. Tindak lanjut adalah tahap terakhir setelah rekomendasi diberikan.
}

::D1_Q09:: Mengapa 'Exit Meeting' (Rapat Penutupan) penting dalam proses audit? {
    =Untuk mendiskusikan temuan awal, mendapatkan persetujuan fakta dari auditee, dan mengklarifikasi kesalahpahaman sebelum laporan final diterbitkan. #Benar. Ini memastikan keakuratan laporan dan membangun hubungan baik.
    ~Untuk menandatangani cek pembayaran jasa auditor. #Salah. Isu pembayaran adalah administratif, bukan tujuan teknis rapat penutupan.
    ~Untuk menentukan siapa yang bertanggung jawab atas kesalahan sistem. #Salah. Audit fokus pada proses dan kontrol, bukan mencari kambing hitam secara personal.
    ~Untuk menyampaikan keputusan pemberhentian karyawan. #Salah. Itu bukan wewenang auditor internal.
}

::D1_Q10:: Seorang auditor menemukan bahwa patch keamanan kritis belum diinstal selama 6 bulan. Temuan ini harus dilaporkan kepada siapa pertama kali? {
    =Manajemen yang bertanggung jawab atas sistem tersebut. #Benar. Temuan dilaporkan ke pemilik aset terlebih dahulu untuk konfirmasi dan rencana perbaikan.
    ~Dewan Direksi. #Salah. Temuan operasional biasanya dilaporkan ke manajemen menengah, kecuali jika itu bencana besar.
    ~Vendor perangkat lunak. #Salah. Auditor melapor ke internal organisasi, bukan vendor.
    ~Seluruh pengguna sistem. #Salah. Ini akan menyebabkan kepanikan dan bukan prosedur pelaporan yang tepat.
}

::D1_Q11:: Apa yang dimaksud dengan 'Materiality' dalam audit sistem informasi? {
    =Tingkat signifikansi kesalahan atau kelalaian yang dapat mempengaruhi keputusan ekonomi pengguna laporan. #Benar. Materialitas menentukan apakah suatu temuan perlu dilaporkan atau tidak.
    ~Jumlah total aset yang diaudit. #Salah. Materialitas adalah ambang batas (threshold), bukan total nilai.
    ~Waktu yang dibutuhkan untuk menyelesaikan audit. #Salah. Ini adalah efisiensi audit, bukan materialitas.
    ~Jumlah staf audit yang terlibat. #Salah. Ini adalah sumber daya.
}

::D1_Q12:: Tujuan dari 'Risk Based Audit' (Audit Berbasis Risiko) adalah... {
    =Memfokuskan sumber daya audit pada area dengan risiko tertinggi. #Benar. Pendekatan ini efisien karena tidak membuang waktu pada area yang risikonya rendah.
    ~Memeriksa semua kontrol tanpa terkecuali. #Salah. Itu adalah audit tradisional/komprehensif yang memakan waktu banyak.
    ~Menghindari area yang berisiko tinggi demi keselamatan auditor. #Salah. Justru area berisiko tinggi harus dikunjungi.
    ~Mengandalkan asuransi untuk menutupi kerugian. #Salah. Ini manajemen risiko transfer, bukan tujuan audit.
}

::D1_Q13:: Bukti audit mana yang dianggap paling dapat diandalkan (Reliable)? {
    =Bukti yang diperoleh auditor secara langsung melalui pengujian atau observasi. #Benar. Bukti primer (first-hand) lebih kuat daripada bukti sekunder.
    ~Pernyataan lisan dari manajemen tanpa dukungan dokumen. #Salah. Pernyataan lisan adalah bukti paling lemah.
    ~Fotokopi dokumen yang disediakan oleh pihak ketiga. #Salah. Fotokopi lebih lemah dari dokumen asli.
    ~Laporan dari audit sebelumnya. #Salah. Kondisi mungkin sudah berubah sejak audit lama.
}

::D1_Q14:: Dalam pengujian kontrol (Test of Controls), jika auditor menemukan bahwa kontrol tidak efektif, langkah apa yang seharusnya diambil? {
    =Melakukan pengujian substantif yang lebih luas untuk mengukur dampaknya. #Benar. Jika kontrol lemah, risiko kesalahan data meningkat, jadi pengujian data harus diperluas.
    ~Menghentikan audit dan melaporkan kegagalan. #Salah. Audit harus tetap dilanjutkan untuk mengukur dampak (substantive testing).
    ~Mengabaikan kontrol dan fokus pada kecepatan audit. #Salah. Tidak profesional.
    ~Meminta manajemen memperbaiki kontrol lalu mengulang audit dari awal. #Salah. Perbaikan kontrol butuh waktu, audit harus tetap diselesaikan dengan pendekatan berbeda.
}

::D1_Q15:: Apa tujuan dari 'Review Phase' (Tinjauan) dalam audit? {
    =Memastikan prosedur audit telah diikuti dan temuan didukung oleh bukti yang memadai. #Benar. Ini adalah Quality Assurance sebelum laporan diterbitkan.
    ~Membayar gaji auditor. #Salah.
    ~Menghapus dokumen kerja audit yang sudah tidak diperlukan. #Salah. Dokumen harus disimpan untuk arsip.
    ~Menentukan fee audit untuk klien. #Salah.
}

::D1_Q16:: Manakah dari berikut ini yang merupakan contoh 'Preventive Control'? {
    =Penggunaan password untuk autentikasi pengguna. #Benar. Password mencegah akses tidak sah sebelum terjadi.
    ~Rekonsiliasi bank bulanan. #Salah. Ini adalah Detective Control (mendeteksi kesalahan setelah terjadi).
    ~Proses pemulihan bencana (Disaster Recovery). #Salah. Ini adalah Corrective Control (memperbaiki setelah insiden).
    ~Laporan aktivitas pengguna (User Activity Log). #Salah. Ini adalah Detective Control.
}

::D1_Q17:: Auditor menemukan bahwa log aktivitas sistem (system logs) tidak diaktifkan. Apa implikasi utama hal ini terhadap audit? {
    =Auditor tidak dapat memverifikasi jejak audit dan harus bergantung pada bukti alternatif atau menganggap risiko tinggi. #Benar. Tidak adanya log menghilangkan kemampuan untuk menelusuri kejadian.
    ~Audit bisa dilakukan lebih cepat karena tidak perlu menganalisis log. #Salah. Ini justru menghambat pengumpulan bukti.
    ~Sistem akan otomatis lebih aman dari serangan. #Salah. Tidak ada hubungannya.
    ~Manajemen pasti menyetujui kondisi ini. #Salah. Asumsi yang tidak berdasar.
}

::D1_Q18:: Dalam standar profesional ISACA, apa kewajiban auditor mengenai 'Due Professional Care'? {
    =Auditor harus melakukan kehati-hatian dan kompetensi yang diharapkan dari seorang profesional yang berpengalaman. #Benar. Ini mencakup kejujuran, objektivitas, dan ketelitian.
    ~Auditor harus menjamin bahwa sistem bebas dari kesalahan. #Salah. Auditor tidak memberikan jaminan (guarantee), hanya memberikan pendapat (opinion).
    ~Auditor harus mengikuti keinginan klien tanpa syarat. #Salah. Auditor harus independen dan taas pada standar.
    ~Auditor harus memiliki sertifikasi CISA sebelum melakukan pekerjaan apa pun. #Salah. Meskipun CISA standar emas, kewajiban 'Due Care' berlaku umum bagi profesional.
}

::D1_Q19:: Apa fungsi 'Working Papers' (Kertas Kerja Audit)? {
    =Mendokumentasikan bukti audit, metode, dan kesimpulan untuk mendukung opini audit. #Benar. Ini adalah catatan resmi yang digunakan untuk review dan bukti hukum.
    ~Sebagai panduan bagi programmer dalam coding. #Salah.
    ~Sebagai surat perjanjian kontrak kerja. #Salah.
    ~Sebagai media komunikasi dengan pelanggan. #Salah.
}

::D1_Q20:: Jika auditor internal menemukan bukti kecurangan (fraud) yang melibatkan manajemen senior, kepada siapa ia harus melapor? {
    =Audit Committee (Komite Audit) atau Dewan Direksi. #Benar. Untuk menjaga independensi dan menghindari konflik kepentingan, pelaporan melampaui manajemen operasional.
    ~Manajer TI yang bersangkutan. #Salah. Pelaku atau atasannya tidak boleh menjadi penerima laporan.
    ~Departemen HRD. #Salah. HRD mungkin terlibat nanti, tapi pelaporan utama ke Komite Audit.
    ~Ke polisi langsung. #Salah. Organisasi biasanya menyelesaikan secara internal dulu sebelum melibatkan hukum, kecuali diwajibkan oleh regulasi.
}

::D1_Q21:: Teknik apa yang digunakan auditor untuk menguji keandalan proses otomatis tanpa menggunakan data langsung? {
    =Test Data (Data Uji). #Benar. Auditor memasukkan data fiktif (dummy) ke dalam sistem untuk melihat apakah sistem memprosesnya dengan benar.
    ~Inquiry (Wawancara). #Salah. Wawancara hanya bertanya, tidak menguji proses sistem secara teknis.
    ~Observation (Pengamatan). #Salah. Pengamatan melihat perilaku staf, bukan logika internal sistem.
    ~Analytical Review (Tinjauan Analitis). #Salah. Ini membandingkan angka/ekpektasi, bukan menguji logika proses.
}

::D1_Q22:: Apa yang dimaksud dengan 'Integrated Audit'? {
    =Audit yang menggabungkan pengujian kontrol keuangan, operasional, dan kepatuhan. #Benar. Pendekatan holistik yang melihat TI terintegrasi dengan bisnis.
    ~Audit yang dilakukan oleh tim gabungan internal dan eksternal. #Salah. Ini adalah Co-sourcing, bukan definisi Integrated Audit.
    ~Audit yang menggunakan software terintegrasi. #Salah. Ini tentang alat, bukan konsep audit.
    ~Audit yang dilakukan setiap hari secara terus-menerus. #Salah. Ini adalah Continuous Auditing.
}

::D1_Q23:: Manakah dari berikut ini adalah keuntungan utama menggunakan 'Generalized Audit Software' (GAS)? {
    =Kemampuan untuk mengakses dan memproses berbagai format data tanpa perlu menulis program khusus. #Benar. GAS seperti ACL/IDEA dirancang agar auditor tidak perlu jadi programmer ahli.
    ~Otomatis menghasilkan laporan audit final. #Salah. Auditor tetap harus menganalisis outputnya.
    ~Dapat mendeteksi semua jenis virus. #Salah. Itu fungsi Antivirus.
    ~Gratis dan tidak memerlukan lisensi. #Salah. Software audit biasanya berbayar.
}

::D1_Q24:: Dalam pemilihan sampel audit, jika auditor mengharapkan tingkat kesalahan yang sangat rendah, apa yang harus dilakukan? {
    =Meningkatkan ukuran sampel untuk memastikan tingkat kepercayaan yang tinggi. #Benar. Jika diasumsikan bersih, auditor perlu sampel lebih banyak untuk membuktikannya.
    ~Mengurangi ukuran sampel karena yakin tidak ada kesalahan. #Salah. Asumsi tidak bisa menggantikan bukti; mengurangi sampel meningkatkan risiko deteksi.
    ~Menghentikan pengujian. #Salah.
    ~Mengambil sampel hanya 1 item. #Salah. Tidak representatif.
}

::D1_Q25:: Apa yang harus dilakukan auditor jika auditee menolak memberikan dokumen yang diperlukan? {
    =Mendokumentasikan penolakan tersebut dan menilai dampaknya terhadap kemampuan auditor dalam memberikan opini (Disclaimer of Opinion mungkin diperlukan). #Benar. Pembatasan ruang lingkup ini serius dan harus dilaporkan.
    ~Memaksa auditee dengan ancaman hukum. #Salah. Auditor internal biasanya melapor ke atasan/audit committee; auditor eksternal melapor ke regulator/klien.
    ~Mengganti dokumen tersebut dengan asumsi auditor. #Salah. Tidak boleh berdasarkan asumsi.
    ~Mengabaikan area tersebut dan melanjutkan audit bagian lain. #Salah. Harus diidentifikasi sebagai batasan (limitation) dalam laporan.
}

::D1_Q26:: Sebuah organisasi menggunakan jasa penyedia layanan eksternal untuk pengelolaan data center. Apa dokumen terbaik yang harus diminta auditor untuk memahami kontrol di pihak ketiga tersebut? {
    =SSAE 18 (SOC 1 Type 2) Report. #Benar. Laporan SOC 1 Type 2 memberikan opini auditor independen atas efektivitas kontrol di organisasi jasa yang relevan dengan pelaporan keuangan.
    ~Kontrak layanan (Service Level Agreement/SLA). #Salah. SLA hanya membahas kinerja (uptime, waktu respons), bukan deskripsi detail kontrol internal.
    ~Kebijakan keamanan pihak ketiga. #Salah. Kebijakan hanya menyatakan rencana, bukan bukti implementasi seperti yang ada di laporan SOC.
    ~Laporan audit internal pihak ketiga. #Salah. Laporan ini mungkin tidak objektif atau tidak mengikuti standar pelaporan yang diakui secara internasional.
}

::D1_Q27:: Manakah dari berikut ini yang merupakan teknik pengambilan sampel (sampling) yang paling tepat untuk menguji apakah password pengguna diubah setiap 90 hari? {
    =Attribute Sampling. #Benar. Attribute sampling digunakan untuk menguji kepatuhan terhadap aturan biner (ya/tidak), misalnya "apakah password sudah diubah?".
    ~Variables Sampling. #Salah. Variables sampling digunakan untuk memperkirakan nilai moneter atau jumlah, seperti total nilai inventaris.
    ~Stop-or-Go Sampling. #Salah. Teknik ini digunakan ketika auditor yakin tingkat kesalahan sangat rendah dan ingin menghentikan pengujian lebih awal.
    ~Discovery Sampling. #Salah. Ini digunakan untuk menemukan fraud yang sangat langka, bukan pengujian kepatuhan rutin.
}

::D1_Q28:: Menurut standar profesional ISACA, apa sikap auditor internal mengenai independensi? {
    =Harus independen dari area yang diaudit, tetapi tetap bertanggung jawab kepada manajemen atau komite audit. #Benar. Independensi berarti tidak ada konflik kepentingan langsung, meskipun tetap bagian dari organisasi.
    ~Harus sepenuhnya independen dari organisasi seperti auditor eksternal. #Salah. Auditor internal adalah karyawan organisasi, independensi mereka adalah fungsional/organisational.
    ~Tidak perlu independen karena auditor internal dipercaya oleh manajemen. #Salah. Independensi adalah prinsip dasar audit yang harus dijaga.
    ~Independensi hanya diperlukan saat menemukan fraud. #Salah. Independensi diperlukan di seluruh proses audit.
}

::D1_Q29:: Bukti audit mana yang dianggap paling kuat untuk membuktikan bahwa sebuah firewall berfungsi sesuai konfigurasi yang disetujui? {
    =Screenshot hasil pengujian penetrasi atau konfigurasi yang diambil langsung oleh auditor. #Benar. Bukti yang diperoleh langsung oleh auditor (inspeksi/observasi) memiliki keandalan tertinggi.
    ~Konfirmasi lisan dari Administrator Jaringan. #Salah. Bukti lisan adalah yang paling lemah karena subjektif dan sulit diverifikasi.
    ~Salinan dokumen kebijakan firewall yang disahkan manajemen. #Salah. Dokumen kebijakan adalah bukti bahwa aturan ada, bukan bukti bahwa aturan diterapkan di sistem.
    ~Laporan review pihak ketiga tahun lalu. #Salah. Kondisi mungkin sudah berubah sejak tahun lalu.
}

::D1_Q30:: Apa langkah pertama yang harus dilakukan auditor dalam melakukan 'Risk Assessment' (Penilaian Risiko) untuk perencanaan audit? {
    =Mengidentifikasi aset TI dan proses bisnis yang kritis. #Benar. Anda tidak bisa menilai risiko jika tidak tahu apa saja aset yang dimiliki (identifikasi aset).
    ~Menghitung probabilitas ancaman. #Salah. Ini langkah selanjutnya setelah aset diidentifikasi.
    ~Memilih area audit yang paling mudah. #Salah. Ini bukan pendekatan berbasis risiko.
    ~Meminta daftar risiko dari manajemen. #Salah. Auditor harus melakukan penilaian sendiri secara independen, meski masukan manajemen diperlukan.
}

::D1_Q31:: Apa yang dimaksud dengan 'Control Self-Assessment' (CSA)? {
    =Proses di mana staf operasional menilai efektivitas kontrol internal mereka sendiri dengan fasilitasi dari auditor. #Benar. CSA melibatkan pemilik proses untuk aktif dalam evaluasi risiko dan kontrol.
    ~Audit yang dilakukan oleh staf internal tanpa sertifikasi. #Salah. CSA adalah metodologi, bukan status auditor.
    ~Audit yang berfokus pada diri sendiri (self-audit) tanpa melibatkan orang lain. #Salah. CSA biasanya dilakukan dalam workshop tim.
    ~Evaluasi oleh auditor terhadap moral staf. #Salah. Ini tidak relevan.
}

::D1_Q32:: Dalam formula Audit Risk = Inherent Risk × Control Risk × Detection Risk, jika auditor ingin menurunkan Detection Risk, apa yang harus dilakukan? {
    =Meningkatkan intensitas dan luasnya pengujian substantif. #Benar. Detection Risk dikendalikan oleh auditor; semakin banyak tes, semakin kecil risiko gagal mendeteksi kesalahan.
    ~Meningkatkan ketergantungan pada kontrol internal. #Salah. Ini menurunkan Control Risk, bukan secara langsung Detection Risk (meski berhubungan terbalik).
    ~Mengurangi jumlah sampel. #Salah. Ini justru meningkatkan Detection Risk.
    ~Meminta manajemen memperbaiki sistem. #Salah. Ini mengurangi Inherent/Control Risk.
}

::D1_Q33:: Teknik audit mana yang paling efektif untuk memverifikasi bahwa proses transfer dana antar bank tidak terjadi duplikasi? {
    =Computer Assisted Audit Techniques (CAATs) untuk membandingkan nomor referensi transaksi dan mencari duplikat. #Benar. CAATs dapat memindai seluruh populasi data untuk menemukan duplikasi yang tidak terlihat secara manual.
    ~Wawancara dengan kasir. #Salah. Manusia tidak bisa mengingat ribaran transaksi.
    ~Pengamatan di lantai kerja. #Salah. Proses transfer elektronik tidak bisa diamati secara visual.
    ~Mengecek rekonsiliasi bank. #Salah. Rekonsiliasi memastikan saldo benar, tetapi mungkin tidak mendeteksi duplikasi jika nilainya seimbang.
}

::D1_Q34:: Seorang auditor IS menemukan bahwa backup data hanya disimpan di lokasi yang sama dengan server utama. Temuan ini menunjukkan kelemahan pada... {

    =Malware yang bersembunyi di dalam sistem operasi untuk memberikan akses tersembunyi (backdoor) kepada penyerang. #Benar. Rootkit sangat sulit dideteksi karena ia menginfeksi level kernel/OS dan bisa menyembunyikan prosesnya sendiri dari daftar task manager.
    ~Virus yang menyerang file root Linux saja. #Salah. Bisa menyerang Windows juga.
    ~Software untuk memperbaiki partisi root. #Salah.
    ~Tools untuk membangun jaringan. #Salah.
}

::D5_Q83:: Apa risiko utama dari serangan *XSS* (Cross-Site Scripting)? {
    =Penyerang dapat menyuntikkan script berbahaya ke halaman web yang dilihat korban, untuk mencuri cookie sesi atau mendeface website. #Benar. XSS menyerang *pengguna* website, bukan server langsung. Script berjalan di browser korban.
    ~Penyerang mendapatkan akses root ke server. #Salah. Itu biasanya OS Command Injection.
    ~Server website menjadi hang karena CPU naik. #Salah. Itu biasanya DoS.

}

::D5_Q84:: Apa tujuan dari *Privileged Access Management* (PAM)? {
    =Mengontrol dan memantau akses akun dengan hak istimewa tinggi (admin, root) untuk mencegah penyalahgunaan. #Benar. PAM tools biasanya menyediakan *password vaulting* (kredensial admin disimpan aman), *session recording* (rekam aktivitas admin), dan *just-in-time access* (akses sementara).
    ~Memberikan akses istimewa ke semua karyawan. #Salah.
    ~Memblokir akses internet untuk admin. #Salah.
    ~Mengelola password user biasa. #Salah. Itu Password Manager/SSO umum.
}

    ~Mengirim virus melalui email. #Salah.
    ~Membuat tanda tangan digital. #Salah.

::D5_Q88:: Apa yang dimaksud dengan *Trojan Horse*? {

    ~Virus yang menyebar otomatis tanpa bantuan user. #Salah. Itu Worm.
    ~Program yang merekam ketikan keyboard. #Salah. Itu Keylogger (bisa jadi bagian dari Trojan).

}

::D5_Q92:: Apa fungsi dari *Degaussing*? {
    =Menghapus data pada media magnetis (Hard Disk, Tape) dengan menggunakan medan magnet yang kuat. #Benar. Degaussing mengubah polaritas magnetis media sehingga data hilang total. Ini merusak media sehingga tidak bisa dipakai lagi (untuk HDD modern, SSD tidak bisa di-degauss).
    ~Membersihkan virus dari memori. #Salah.
    ~Menghapus file sampah. #Salah.
    ~Memformat disk secara logis. #Salah. Format tidak seaman degaussing.
}
